Έστω μια μεγάλη εταιρεία λογισμικού η οποία έχει κάποια λεφτά τα οποία για κάποιο λόγο θέλει να τα διαθέσει για Open Source Λογισμικό. Για το "τεχνολογικό" οικοσύστημά της(π.χ. java,javascript,python κλπ.), πώς θα πρέπει να κάνει αυτήν την χρηματοδότηση σε developers προκειμένου συνολικά να συνεχίσει να ευδοκημεί το οικοσύστημα? 

Ανάλογα με τον package manager(σύστημα διανομής βιβλιοθηκών λογισμικού) του οικοσυστήματος μπορούμε να δούμε ποιοι είναι οι πιο πολύτιμοι(influential) developers με βάση κάποια social features μεταξύ των developers στο software development process. Ενδεικτικά θα μπορούσαμε να χρησιμοποιήσουμε(με διαφορετικά βάρη εννοείται):
- Το ποιος κάνει code reviews τακτικά σε ποιους?
- Ποιοι συμμετέχουν σε επιτυχημένα(merged) PRs σε πρότζεκτ.
- Ποιος συμμετέχει σε συζητήσεις στα issues (είτε closed είτε resolved)

Σημαντική λεπτομέρεια: Δεν βελτιστοποιούμε για το effectiveness κάποιου μεμονωμένου developer άμα τον πληρώνουμε να δουλέψει σε κάποιο πρότζεκτ, αλλά ποιους developers αν επιβραβεύσουμε για τα contributions, θα κινηθεί περισσότερο το κοινωνικό γρανάζι του Open Source και τελικά το ίδιο το Open Source. Το Open Source είναι μια κοινωνικοτεχνική διαδικασία, και το τεχνικό κομμάτι είναι κάτι που πειράζεται με πολύ διαφορετικές μεθόδους.
